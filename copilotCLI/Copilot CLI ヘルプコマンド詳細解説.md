# GitHub Copilot CLI ヘルプコマンド詳細解説

GitHub Copilot CLIの`/help`コマンドで表示される機能の詳細な日本語解説です。

## ? グローバルショートカット

### ファイル操作
- **`@`** - **ファイルメンション機能**
  - ファイルを参照し、その内容を現在のコンテキストに含める
  - 使用例: `@README.md` でREADMEファイルの内容をAIが参照可能
  - コードレビューや説明時に特定ファイルを参照する際に便利

### 操作制御
- **`Esc`** - **現在の操作をキャンセル**
  - 実行中の処理を中断
  - 入力中の内容をリセット

- **`!`** - **ローカルシェル直接実行**
  - Copilotを経由せずに、コマンドを直接ローカルシェルで実行
  - 使用例: `!ls` でディレクトリ一覧を直接表示
  - AIの解釈なしでシステムコマンドを実行したい場合に使用

### システム操作
- **`Ctrl+c`** - **マルチファンクション操作**
  - 思考中の場合：操作をキャンセル
  - 入力がある場合：入力内容をクリア
  - 何もない場合：CLIを終了

- **`Ctrl+d`** - **シャットダウン**
  - Copilot CLIを完全に終了

- **`Ctrl+l`** - **画面クリア**
  - ターミナル画面をクリアして見やすくする

## ? タイムライン表示制御ショートカット

### 表示切り替え
- **`Ctrl+o`** - **全タイムライン展開/折りたたみ**
  - すべての過去の操作履歴を展開表示
  - 再度押すと折りたたみ表示に戻る
  - プロジェクト全体の作業履歴を確認したい場合に使用

- **`Ctrl+r`** - **最近のタイムライン展開/折りたたみ**
  - 最近の操作履歴のみを展開表示
  - 直近の作業内容を素早く確認したい場合に便利

## ?? カーソル移動・編集ショートカット

### 行内移動
- **`Ctrl+a`** - **行の始まりに移動**
  - カーソルを現在行の最初の文字に移動
  - Bashライクなキーバインド

- **`Ctrl+e`** - **行の終わりに移動**
  - カーソルを現在行の最後の文字に移動

### 文字・単語削除
- **`Ctrl+h`** - **前の文字を削除**
  - Backspaceキーと同等の機能

- **`Ctrl+w`** - **前の単語を削除**
  - カーソル位置から前の単語境界まで削除
  - 単語単位での素早い編集に便利

### 行削除
- **`Ctrl+u`** - **カーソルから行頭まで削除**
  - 現在のカーソル位置から行の始まりまでのテキストを削除

- **`Ctrl+k`** - **カーソルから行末まで削除**
  - 現在のカーソル位置から行の終わりまでのテキストを削除

### 単語移動
- **`Meta+←/→`** - **単語単位でカーソル移動**
  - 左右の矢印キーとMetaキー（Alt/Option）の組み合わせ
  - 単語境界でカーソルをジャンプ移動

## ? 履歴機能

### コマンド履歴
- **`↑↓`キー** - **コマンド履歴のナビゲーション**
  - 上矢印：過去のコマンドに戻る
  - 下矢印：新しいコマンドに進む
  - 過去に実行したプロンプトを再利用可能

## ? 指示ファイルサポート

Copilot CLIは以下の場所から指示ファイルを自動的に読み込み、AIの動作をカスタマイズします：

### プロジェクト固有の指示
- **`.github/instructions/**/*.instructions.md`**
  - GitHubリポジトリ内の詳細な指示ファイル
  - 複数のファイルに分けて整理可能

- **`.github/copilot-instructions.md`**
  - プロジェクト全体に適用される基本指示

- **`**/AGENTS.md`**
  - AIエージェント向けの汎用指示ファイル
  - プロジェクト内のどこにでも配置可能

### AI特化の指示
- **`CLAUDE.md`**
  - Claude専用の指示ファイル
  - Claudeモデル使用時に適用

- **`GEMINI.md`**
  - Gemini専用の指示ファイル
  - Geminiモデル使用時に適用

### ユーザー固有の指示
- **`$HOME/.copilot/copilot-instructions.md`**
  - ユーザーのホームディレクトリに配置
  - 全プロジェクト共通の個人設定

## ? 利用可能なスラッシュコマンド

### ディレクトリ・アクセス管理
- **`/add-dir <directory>`** - **ディレクトリアクセス許可追加**
  - 指定したディレクトリをファイルアクセス許可リストに追加
  - セキュリティ上の理由で、明示的に許可したディレクトリのみアクセス可能

- **`/list-dirs`** - **許可ディレクトリ一覧表示**
  - 現在アクセス許可されているディレクトリの一覧を表示
  - セキュリティ設定の確認に使用

- **`/cwd [directory]`** - **作業ディレクトリ操作**
  - 引数なし：現在の作業ディレクトリを表示
  - 引数あり：指定したディレクトリに変更
  - プロジェクト間の移動に便利

### セッション管理
- **`/clear`** - **会話履歴クリア**
  - 現在のセッションの会話履歴を削除
  - 新しいトピックで作業を開始する際に使用

- **`/session`** - **セッション情報表示**
  - 現在のCLIセッションに関する詳細情報を表示
  - セッション時間、使用統計などを確認

- **`/exit`** - **CLI終了**
  - GitHub Copilot CLIを終了
  - `Ctrl+d`と同じ機能

### 認証・ユーザー管理
- **`/login`** - **Copilotにログイン**
  - GitHubアカウントでCopilotサービスにログイン
  - 初回使用時や認証が切れた際に使用

- **`/logout`** - **Copilotからログアウト**
  - 現在のCopilotセッションからログアウト
  - セキュリティやアカウント切り替え時に使用

- **`/user [show|list|switch]`** - **GitHubユーザー管理**
  - `show`：現在のユーザー情報を表示
  - `list`：利用可能なユーザーアカウント一覧
  - `switch`：他のGitHubアカウントに切り替え

### AI・モデル設定
- **`/model [model]`** - **AIモデル選択**
  - 引数なし：利用可能なモデル一覧を表示
  - 引数あり：指定したモデルに切り替え
  - 対応モデル：Claude Sonnet 4.5、Claude Sonnet 4、GPT-5

- **`/reset-allowed-tools`** - **許可ツールリストリセット**
  - セキュリティ設定で許可されたツールのリストを初期化
  - 権限設定をクリーンな状態に戻す

### MCP（Model Context Protocol）管理
- **`/mcp [show|add|edit|delete|disable|enable] [server-name]`** - **MCPサーバー設定管理**
  - `show`：現在のMCPサーバー設定を表示
  - `add`：新しいMCPサーバーを追加
  - `edit`：既存のMCPサーバー設定を編集
  - `delete`：MCPサーバーを削除
  - `disable/enable`：MCPサーバーを無効化/有効化
  - GitHub MCPサーバーがデフォルトで利用可能

### UI・表示設定
- **`/theme [show|set|list] [auto|dark|light]`** - **ターミナルテーマ設定**
  - `show`：現在のテーマ設定を表示
  - `set`：テーマを設定（auto/dark/light）
  - `list`：利用可能なテーマ一覧
  - 視認性や好みに応じてカロールschemeを調整

### 統計・フィードバック
- **`/usage`** - **セッション使用統計表示**
  - プレミアムリクエスト使用量
  - セッション時間
  - コード変更統計
  - モデル別トークン使用量

- **`/feedback`** - **フィードバック送信**
  - GitHub Copilot CLI開発チームへのフィードバック送信
  - 機能改善やバグレポートに活用

- **`/help`** - **ヘルプ表示**
  - インタラクティブコマンドのヘルプを表示
  - 本ドキュメントの元となった情報

## ? 使用のコツとベストプラクティス

### 効率的な操作方法
1. **ファイルメンション（`@`）の活用**
   - コードレビュー時に関連ファイルを直接参照
   - 複数ファイルの関係性を説明する際に便利

2. **ショートカットキーの習得**
   - `Ctrl+r`で最近の作業履歴を素早く確認
   - `Ctrl+l`で画面をクリーンに保つ

3. **指示ファイルの活用**
   - プロジェクト固有のコーディング規約を`.github/copilot-instructions.md`に記載
   - チーム共通のベストプラクティスを共有

### セキュリティ考慮事項
- `/add-dir`で必要最小限のディレクトリのみ許可
- 機密情報を含むディレクトリへのアクセスは慎重に管理
- 定期的に`/list-dirs`で許可設定を確認

### トラブルシューティング
- 認証エラー時は`/logout`後に`/login`で再認証
- 応答が遅い場合は`/model`で軽量なモデルに切り替え
- セッションが重い場合は`/clear`で履歴をクリア
